# Learning Journal

This file is here to help when your thoughts are messy.
It is not supposed to sound formal.

If a topic gets large, move it into a separate `day_0X.md` file.
Use this file as the default place for rough thinking, definitions, and reminders.

## What a useful note looks like

A useful note usually answers these:

- What did I work on?
- What was confusing at first?
- What do I now understand better?
- Why did I use this method or library?
- What still feels weak?
- What do I need to do next?

If the note does not answer those, it is probably too vague.

## Template

### Topic
- What am I trying to understand?

### What this is in plain language
- Explain it simply.
- If you cannot explain it simply, that is a signal.

### Why it matters in this project
- What part of the pipeline depends on this?

### What I used
- dataset
- code file
- notebook
- model
- library

### Why I used it
- Why this was the right choice for now
- What I could have used instead
- Why I did not use the alternative yet

### What I learned
- What is clearer now

### What still feels shaky
- Questions I still cannot answer cleanly

### Mistakes or traps
- What I got wrong
- What would be easy to get wrong again

### Next move
- What I should do immediately after this
